# Distributed Systems

## Overview

This section covers service boundaries, asynchronous messaging, Kafka, event-driven architecture, delivery guarantees, resiliency, and the operational realities of systems that fail across process and network boundaries.

## Folder Map

- `fundamentals/`: core distributed systems concepts and failure modes
- `interview/`: interview-oriented questions on services, messaging, and reliability
- `examples/`: patterns such as outbox, saga coordination, and event versioning
